⚙️ Supabase Database Setup

Run this SQL once in your Supabase SQL Editor to create both tables (posts + subscribers):

-- POSTS TABLE CREATE TABLE IF NOT EXISTS cc_posts ( id bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY, title text NOT NULL, body text NOT NULL, author text NOT NULL, cat text NOT NULL, read_time int NOT NULL DEFAULT 5, created_at timestamptz DEFAULT now() ); -- SUBSCRIBERS TABLE CREATE TABLE IF NOT EXISTS cc_subscribers ( id bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY, name text NOT NULL, email text NOT NULL UNIQUE, created_at timestamptz DEFAULT now() ); -- SECURITY POLICIES ALTER TABLE cc_posts ENABLE ROW LEVEL SECURITY; ALTER TABLE cc_subscribers ENABLE ROW LEVEL SECURITY; CREATE POLICY "posts_read" ON cc_posts FOR SELECT USING (true); CREATE POLICY "posts_insert" ON cc_posts FOR INSERT WITH CHECK (true); CREATE POLICY "posts_delete" ON cc_posts FOR DELETE USING (true); CREATE POLICY "posts_update" ON cc_posts FOR UPDATE USING (true); CREATE POLICY "subs_read" ON cc_subscribers FOR SELECT USING (true); CREATE POLICY "subs_insert" ON cc_subscribers FOR INSERT WITH CHECK (true); CREATE POLICY "subs_delete" ON cc_subscribers FOR DELETE USING (true); -- SETTINGS TABLE CREATE TABLE IF NOT EXISTS cc_settings ( key text PRIMARY KEY, value text NOT NULL, updated_at timestamptz DEFAULT now() ); ALTER TABLE cc_settings ENABLE ROW LEVEL SECURITY; CREATE POLICY "settings_read" ON cc_settings FOR SELECT USING (true); CREATE POLICY "settings_insert" ON cc_settings FOR INSERT WITH CHECK (true); CREATE POLICY "settings_update" ON cc_settings FOR UPDATE USING (true);
  1. Go to supabase.com → your project → SQL Editor → paste & run the SQL above.
  2. Go to Project Settings → API. Copy your Project URL and anon/publishable key.
  3. Open index.html in a text editor. Find the SUPABASE_URL and SUPABASE_KEY variables at the top and replace them.
  4. For GitHub Pages: Go to github.com → New Repository → name it clauseandcapital → upload index.html → Settings → Pages → Deploy from main branch → your site is live at https://yourusername.github.io/clauseandcapital
  5. Optionally change ADMIN_PASS from cc2026 to your own password.
  6. The Curated tab uses cc_settings. Run the updated SQL above once so curated edits can be saved.
